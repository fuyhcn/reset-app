# Reset（正是修行时）V1 交付评估报告

> 评估范围：基于 `reset-app/` 源码、已完成的 30+ 轮迭代记录、产品文档《Reset Design Bible v1.0》、以及已交付的纯逻辑单测 `qa/test.mjs`。
> 评估维度：测试、UI 设计、文档还原度、V2 规划。

---

## 一、测试情况

### 现状盘点

| 测试类型 | 现状 | 说明 |
|---------|------|------|
| 纯逻辑单测 | ✅ 已存在 | `qa/test.mjs`，覆盖 store / 备份纯逻辑 |
| 真机/用户验收 | ✅ 已通过 | 你本人已上机使用，反馈"功能都 ok" |
| 构建门禁 | ✅ 已存在 | `npm run build` = `vue-tsc -b && vite build`，类型零错误 |
| 自动化测试门禁 | ❌ 缺失 | `package.json` 无 `test` 脚本，单测需手工 tsc 编译 + 别名 loader 才能跑 |
| 组件 / UI 测试 | ❌ 缺失 | 无 Vitest + @vue/test-utils |
| CI / 回归 | ❌ 缺失 | 无持续集成 |

### 单测覆盖质量（诚实评价：偏上）
`qa/test.mjs` 不是凑数的"hello world"，而是按**边界驱动**写的，包含：
- `settingsStore`：打卡连续天数（断签、今天未打卡、跨月/跨年 41 天）、`checkinToday` 幂等；
- `eventStore`：8 类 action 映射、summary 的 `smokeCount`（smoked+urge_failed 语义）、`exerciseMinutes` 累加、睡眠时长；
- **连续自控天数 `controlDays` 的 off-by-one 专项用例**（今天抽烟归零、今天空/昨天控制计入等 6 组）；
- `replaceEvents` 导入还原、`useBackup.importData` 三重校验（非 Reset 文件 / 缺 events 均返回 null）。

### 主要缺口
1. **测试未"通电"**：写好了却没有 `npm test` 入口，每次改完靠人工回忆跑，极易回归。建议加一个 `vitest` 或 `node --import tsx` 脚本，把 `qa/test.mjs` 接进去。
2. **无 UI 回归**：动画、Sheet、Dock 毛玻璃等只能靠你真机肉眼验收，无法防"改 A 坏 B"。
3. **无端到端**：导入→导出闭环、睡眠跨天配对等只在逻辑层验证，未覆盖真实文件读写。

**结论**：核心算法层测试扎实，但**工程化测试门禁为 0**，属于"会写但没挂上"。这是 V1 最该补的一块地基。

## 二、UI 设计评估

### 高度契合 Design Bible 的部分（设计完成度高）
- **色彩系统**：主色 Reset Green `#34C759`、Smoke Gray `#8E8E93`、Warning Orange `#FF9500`、Negative Red `#FF3B30` 全部按文档取色；背景 Light `#F2F2F7` / Dark `#000000` 一致。
- **深色模式**：完整支持，且按文档"用户多在夜间打开"的场景做了专门适配（安全区、毛玻璃在深色下也成立）。
- **毛玻璃 Dock**：`QuickDock` 用 `backdrop-filter: blur()` 实现 iOS Dock 质感，符合 4.5 规范。
- **底部 Sheet + Spring 动效**：所有记录弹层均为底部升起 + 弹簧动画，禁用花哨/粒子（符合 2.13）。
- **"不评价用户"原则**：全程正向措辞（"已控制""你成功控制了 X 次"），无打卡红点、无排行榜、无红色失败页（3.15 明令禁止项均未出现）。
- **极简留白 + 圆角 + 8px 间距**：卡片 16–24px 圆角、间距走 8/12/16/24 体系，无 13/17/27 这类乱值。
- **图标系统**：用 Phosphor 图标（`i-ph-*`）替代 emoji，符合"像系统 App 而非聊天"的要求。

### 可提升点（不影响使用，但离"Apple Health 遗失的一部分"还差一口气）
1. **恢复页主视觉**：文档 4.11 要求一个**大数字恢复值（如 86）+ "我正在变好"**。当前 `RecoveryView` 是"恢复指数"环形 + 四模块，方向对，但缺少文档强调的"大数字 + 情绪化文案"冲击力。
2. **撤销机制**：`BaseToast` 已支持 `undoAction`（5 秒撤销），符合 2.11，但需确认每个快速操作都接了它（烟瘾误触尤其需要）。
3. **时间轴布局**：文档要求"左侧时间(55px) + 右侧事件 + 中间细线"（3.10/4.4）。当前 `EventTimeline` 是分组卡片式（`journey-card`），可读性 OK，但和文档的"Apple Timeline 细线"视觉有差异——属于合理演进，非硬伤。
4. **手势**：文档 2.12 的"左滑编辑 / 右滑复制"未实现（当前是点击进详情）。
5. **Haptic**：文档 2.13 的轻震反馈，Web 端暂不可做（需原生壳）。

**结论**：UI 在**设计语言一致性**上得分很高（≥90% 还原了 Calm/Minimal/Personal 的调性），属于"明显受过 Apple HIG 训练的产物"，不是拼凑的。细节上恢复页主视觉和手势是 V2 的打磨点。

## 三、对比文档还原度

### 高度还原
| 维度 | 文档要求 | 实现 |
|------|---------|------|
| 产品定位 | 记录改变过程，非记录生活 | ✅ 事件流驱动 |
| 数据归属 | 不登录、不上传、本地保存 | ✅ localStorage，纯本地 |
| 2 秒记录 | 平均 ≤3 秒 | ✅ 一键 Dock + 二级 Sheet |
| 记录欲望 > 结果 | 记烟瘾 7 次而非"今天抽 5 根" | ✅ urge/smoked 分离，行为链 |
| 默认少输入 | 必填自动生成，选填可选 | ✅ 时间/类型自动，原因/备注可选 |
| 不评价用户 | 禁惩罚性文案 | ✅ 全面正向 |
| 快速 Dock | 毛玻璃、4 高频 + ＋ | ✅ |
| Sheet 交互 | 底部升起、Spring | ✅ |
| 导出 | 生成 JSON、可拼接提示词 | ✅（本次迭代新增下载+导入） |
| 事件枚举 | smoke/exercise/sexual/sleep/mood/learning/weight | ✅ 全部覆盖 |

### 有意偏离（合理）
| 文档 | 实际 | 评价 |
|------|------|------|
| 技术栈含 IndexedDB | 改用 localStorage | ✅ 此规模 IndexedDB 过重，且曾因 Dexie 出过问题，偏离正确 |
| V1 仅 Today + Recovery 两页 | 演进为多页（含睡眠/运动/抽烟/心情/性/体重/提醒/导出/关于） | ✅ 自然生长，用户接受度高 |
| 导出范围"本周/本月/全部" | 仅"本周/全部" | ⚠️ 小缺口，补"本月"即可 |

### 未实现（值得在 V2 补）
| 文档 | 状态 |
|------|------|
| 行为链 `session_id`（5.4） | ❌ urge→smoked 未用 session_id 关联 |
| 睡眠目标 `bed_before`（5.6） | ❌ 仅有"睡眠充足 7h"隐含目标，无就寝时间目标 |
| 手势 左滑/右滑（2.12） | ❌ |
| 分享图/卡片 | ❌ |

**整体还原度评分**：核心交互与数据模型 **≈95%**，视觉规范 **≈90%**，文档 5.x 数据架构高级特性 **≈70%**。对一个边做边改、已上线的 V1 来说，还原度属于优秀档。

## 四、V2 应该做哪些功能

按 Design Bible 的路线图（第二阶段"生活提升"、第三阶段"个人健康数据库 + AI 分析"）和你已暴露的真实痛点（怕丢数据、想要更省心的备份），建议优先级如下：

### 🔴 P0（先把 V1 的地基补完）
1. **测试门禁通电**：`npm test` 接入 `qa/test.mjs`（vitest 或 tsx + alias），保证每次提交不回归。这是"能用"到"敢迭代"的分水岭。
2. **本地自动快照备份**：按上次评估结论，替代"定时写文件"——App 打开时若距上次快照 >1 天，自动存一份到 localStorage（保留近 7 份），导出页可回滚。彻底解决"时间长怕丢数据"，且不踩 iOS 后台限制。

### 🟠 P1（产品竞争力）
3. **AI 分析闭环**：导出已能拼提示词，V2 应让用户在 App 内直接把事件流发给 LLM，返回"你的烟瘾高发时段/诱因分布/睡眠趋势"等洞察。这是文档"数据交给 AI"的终极形态。
4. **行为链 `session_id`**：把一次"想抽→忍住/没忍住"串成链，AI 分析才有"过程"可言（文档 5.4 明确要求）。
5. **恢复页主视觉升级**：大数字恢复值 + "我正在变好"情绪化文案（对齐 4.11）。
6. **更多记录类型（第二阶段）**：饮食、咖啡、酒精、血压——扩展事件枚举，复用现有 Sheet 架构成本低。
7. **睡眠目标 `bed_before`**：增加"建议 23:30 前睡"目标，Recovery 睡眠维度从"时长"升级到"时长 + 就寝纪律"。

### 🟡 P2（打磨与惊喜）
8. **手势**：时间轴左滑编辑 / 右滑复制（2.12），大幅提升老用户效率。
9. **撤销全覆盖**：确保每个快速记录都带 5 秒撤销。
10. **趋势图表**：体重、情绪、吸烟的周/月趋势曲线（Recovery 已有睡眠柱状图，可复用）。
11. **分享卡片**：把"本周恢复报告"生成图片，便于社交分享（注意：不违反"非社交 App"——是分享自己的成长，不是排行榜）。
12. **原生壳（Capacitor）**：若想上架 App Store / 支持 Haptic / 桌面 Widget，需要一层原生包装，但会偏离"纯 PWA 不上传"的初心，建议作为可选分支。

## 五、总评

**V1 是一个"设计驱动、落地扎实"的成品**：产品理念（事件流、不评价、本地优先）被完整贯彻，UI 调性与 Apple HIG 高度一致，核心逻辑有真实单测背书，你已真机验证可用。它已经不是一个 demo，而是一个能每天用的工具。

**唯一的结构性短板是"工程化测试门禁缺失"**——功能对，但改起来心里没底。V2 的第一笔投入应该花在这里，而不是急着加功能。补完测试地基 + 本地快照后，再上 AI 分析和第二阶段记录类型，产品的护城河就立住了。

> 注：导出范围"本月"、行为链 session_id、睡眠 bed_before 目标 这三个是低成本高性价比的文档补全项，可顺手在 V2 早期一并做了。
