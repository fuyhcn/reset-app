# Reset App 增量重构 PRD（简单版）

> 版本：v0.1（增量重构）｜ 角色：产品经理 许清楚 ｜ 语言：中文
> 适用技术栈：Vue3 + Vite + TypeScript + Pinia + UnoCSS + PWA；本次新增 vue-router@5.2.0

## 0. 背景与范围（原始需求复述）
Reset 是本地优先的个人恢复记录系统，核心理念是记录"改变自己的过程"，明确反对游戏化（连续签到火苗、等级、排名、红色失败页）。现状存在三处待重构点：(1) `App.vue` 仅渲染 `TodayView`，无路由（vue-router 已装未接入）；(2) 记录交互均为底部 Sheet 弹窗（`ActionSheet` / `ExerciseSheet` / `MoreSheet`）；(3) 数据层双套不一致——`eventStore`（localStorage，key `reset-events`）是唯一有数据的层，`db`（Dexie/IndexedDB）几乎为空且导出功能失效。本次重构目标：将弹窗改为独立页面 + 多页面导航，新增打卡与系统设置，并修正导出逻辑。

## 1. 产品目标（本次重构）
在不背离"非游戏化恢复记录"核心理念的前提下，将 Reset 从"单页 + 底部弹窗"的交互模型，重构为基于 vue-router 的多页面导航体系；把抽烟 / 运动 / 心情 / 性冲动 / 睡眠等记录从 Sheet 弹窗迁移为独立页面，并以 Today 为首页、More 为枢纽页统一入口；同时补齐克制式的"打卡"与"系统设置（主题切换 / 导出导入 / 关于作者）"，修正数据导出逻辑使其导出 `eventStore` 中的真实事件，从而提升记录专注度、导航清晰度与数据自主权。

## 2. 用户故事
1. 作为戒烟 / 恢复中的用户，我希望把"抽烟记录"从底部弹窗改为独立页面，以便更专注地记录"抽了一根 / 特别想抽 / 没抽"，并补充诱因标签与备注。
2. 作为运动记录用户，我希望在独立运动页面选择运动类型并填写时长与感受，以便清晰归档当天的锻炼。
3. 作为关注情绪的用户，我希望在独立心情页面快速记录当前心情，以便回顾情绪变化。
4. 作为记录隐私维度的用户，我希望在独立页面记录性冲动（正常 / 冲动控制 / 冲动），以便保持该维度的连续性。
5. 作为规律作息的用户，我希望在独立睡眠页面记录睡觉与起床，以便准确统计睡眠时长。
6. 作为日常使用者，我希望在 Today 通过底部 Dock 快速进入各记录页、通过右上角 `•••` 进入 More 枢纽页并跳转到 Recovery / Export / Settings / About / CheckIn，以便统一导航、各页用返回栏回到 Today。
7. 作为希望保持"在场"的用户，我希望对今天打卡（标记为已记录 / 在场）并看到连续天数作为正向提示，以便在无焦虑的前提下温和地坚持记录。
8. 作为注重体验与数据主权的用户，我希望在设置页切换浅色 / 深色 / 跟随系统主题、导出 / 导入真实事件数据、查看关于作者，以便个性化界面并掌握自己的数据。

优先级：P0 必须（本次交付）｜ P1 重要（紧接迭代）｜ P2 后续

## 5. 待确认问题
1. **打卡规则**：是否"每天只能打一次"？漏打是否允许补打？连续在场天数口径是否与现有 `controlDays`（基于 `urge_resisted` 的连续自控天数）合并还是独立统计？（建议：打卡独立记录"在场"日期、连续在场天数单独统计、每天至多一次、不支持补打过去日期——需拍板）
2. **Recovery 页指标范围**：现有 `summary` 仅有 `smokeCount` / `exerciseMinutes` / `sleepHours` / `controlDays`。Recovery 页是否还需抽烟趋势曲线、运动周汇总、心情分布等？指标范围决定 P1（R-20）工作量。
3. **设置 / 导出入口组织**：用户原始诉求把"导出"放在系统设置下，而设计决策拆出独立 `/export` 页。最终是否 `/settings` 仅做模式切换、`/export` 独立负责导出导入、`/about` 独立负责关于？需确认三者在 More 中的组织方式与是否保留"设置内含导出"的入口。
